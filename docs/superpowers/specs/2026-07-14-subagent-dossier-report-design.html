Sub-agent dossier report design

Status: Approved in conversation on 14 July 2026.

Purpose

Replace the generic evaluation with a detailed, visually structured account of every sub-agent used during the Nocturne Studio redesign. The report must explain how each agent was assigned, how well it performed, where it failed, and how future assignments should improve.

Information architecture

  1. Editorial cover with current pause status and verified headline metrics.
  2. Workflow map showing discovery, implementation, independent review, remediation, and root integration.
  3. Agent roster separating Luna discovery work, Terra implementation work, Terra review work, and root-only responsibilities.
  4. One dossier per agent or agent run, containing assignment, constraints, supplied context, outputs, measured results, strengths, failures, confidence limits, rating, and improvement measures.
  5. Cross-agent visuals for performance, findings by task, productive versus wasted handoffs, and test-suite growth.
  6. Recommended future operating model and unresolved redesign work.

Visual direction

The report will use a restrained Nocturne editorial system rather than generic dashboard cards: warm dark paper, ivory type, amber evidence markers, serif display headings, compact monospaced metadata, dossier dividers, CSS-only charts, a workflow diagram, and print-safe styling. Visuals must explain relationships or comparisons rather than decorate empty space.

Evidence rules

  • Use repository commits, task briefs, task reports, progress ledger, returned reviewer findings, and observed agent status.
  • Label missing or unavailable evidence instead of inventing an outcome.
  • Separate implementation success, reviewer effectiveness, responsiveness, and visual-verification limits.
  • Do not claim Tasks 5–8 are complete.
  • Record the stalled Task 5 reviewer as a failed handoff and the replacement reviewer as a successful recovery.

Dossier scoring

Each dossier will score only dimensions supported by evidence: delivery, defect detection, reliability, responsiveness, and evidence quality. A composite score may be shown, but the underlying dimensions and confidence level must remain visible.

Output

Overwrite reports/subagent-evaluation-report.html with one self-contained responsive HTML file using inline CSS and no external dependencies. Preserve the existing Markdown evaluation as a separate source artifact.
