Managing sub-agents personal skill design

Status: Approved in conversation on 15 July 2026.

Location: ~/.codex/skills/managing-subagents

Purpose

Create an adaptive manager protocol for Codex tasks where sub-agents are already authorized. The skill improves delegation decisions, evidence retention, responsiveness, independent review, remediation, and final verification without duplicating the global AGENTS.md authorization rules.

Package

  • SKILL.md, under 100 lines, containing the core operating protocol.
  • references/contracts.md, containing reusable dispatch, checkpoint, completion, review, and run-assessment contracts.
  • agents/openai.yaml, generated by the official skill-creator scripts.

Protocol

  1. Prefer the root agent unless delegation has a concrete benefit.
  2. Map dependencies and write scopes; run writers concurrently only when independent.
  3. Keep root ownership of integration, shared files, user updates, visual verification, and completion claims.
  4. Dispatch using bounded objectives, exact scope, non-goals, verification, evidence, and checkpoint contracts.
  5. Require cited durable artifacts from discovery agents and proof of work from implementers.
  6. Request one checkpoint after unexplained silence; replace after a missed checkpoint with a narrower package.
  7. Use a fresh read-only reviewer for specification and quality review, then return findings to the original implementer and re-review the full range.
  8. Use cross-boundary checklists derived from the observed SSR, focus, mutation ownership, persistence, ordering, browser input, hydration, timing, live-region, reduced-motion, and visual-quality failures.
  9. Assess delivery, correctness, responsiveness, and evidence quality; do not score missing evidence.

Evidence and validation

The Hindsight redesign supplies baseline failure evidence: transient Luna output, repeated implementer self-review misses, a stalled large-package reviewer, and thirteen concrete review findings across Tasks 1–5. The initial draft will run structural validation only because the user stopped all sub-agents. Forward-testing remains explicitly unverified until separately authorized.
