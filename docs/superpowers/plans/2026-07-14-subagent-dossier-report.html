Sub-agent dossier report implementation plan

Execution: Inline by the root agent. Sub-agents remain stopped.

Goal: Replace the existing generic report with a detailed, visual, evidence-backed dossier for every agent run used during the Nocturne redesign.

Architecture: One dependency-free HTML artifact with a semantic document structure, inline CSS, CSS-only explanatory visuals, and explicit evidence/confidence labels.

Tech: HTML5 and CSS only.

Global constraints

  • Do not restart sub-agents.
  • Do not use external assets, scripts, CDNs, or invented evidence.
  • Preserve the Markdown evaluation and overwrite only reports/subagent-evaluation-report.html.
  • Keep the report responsive, printable, keyboard-readable, and honest about unavailable evidence.

Task 1: Build the verified agent ledger

  1. Read the approved design, progress ledger, task briefs/reports, commit history, and returned review findings.
  2. Record every known Luna and Terra agent run, its assignment, output, measurable result, shortcomings, and evidence confidence.
  3. Mark any unavailable discovery-agent output as unverified instead of inferring details.

Task 2: Rebuild the report

  1. Replace the old page with the approved editorial cover, workflow map, roster, agent dossiers, comparison visuals, failure analysis, and future operating model.
  2. Use CSS-only flow lines, bars, score rings, and task/findings graphics that encode labelled information.
  3. Give each dossier consistent sections for assignment, usage, output, scoring, failures, evidence, and improvement measures.

Task 3: Verify the artifact

  1. Check HTML headings and all named agent dossiers with rg.
  2. Run git diff --check -- reports/subagent-evaluation-report.html; expect exit 0.
  3. Confirm the file is self-contained by checking that it contains no external http:// or https:// references.
  4. Inspect the rendered file locally if browser control is available; otherwise disclose that visual rendering was not automatically inspected.
