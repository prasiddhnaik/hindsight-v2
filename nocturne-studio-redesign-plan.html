Hindsight: Nocturne Studio redesign

A complete dark-first redesign of the chat workspace, interaction states, conversation navigation, tool activity, and memory management—while preserving the existing persistence, streaming, context, and safety architecture.

No production work before approval Prepared 14 July 2026 Next.js 15 · React 19 · TypeScript · Tailwind 4 Runtime: Bun · package manager: pnpm

Agent execution contract

For agentic workers: implementation must use the subagent-driven development workflow after this plan is approved. A fresh coding agent receives one bounded task, the root agent reviews its diff and verification evidence, and only then does the next dependent task begin.

No subagent may delete files. Luna is reserved for low-complexity reading or verification summaries, Terra for substantial code generation, and Sol will not be used. The root agent owns shared-file integration, temporary-file cleanup, final browser verification, and all completion claims.

Current understanding and locked decisions

Selected identity

Nocturne Studio

Cinematic dark surfaces, editorial typography, warm apricot action color, quiet green success states, restrained borders, and purposeful motion.

Scope

Visual system plus UX completion

The work includes multiline input, stop/regenerate, tool details, grouped history, recovery states, accessible mobile navigation, and improved memory controls.

Theme assumption

Dark-first and dark-only in this pass

A light theme is excluded so the selected design remains coherent. Semantic tokens will keep a future light palette possible without restructuring components.

Preserved architecture

Database remains the source of truth

Lazy conversation creation, shallow URL replacement, streaming, context budgets, compaction, long-term memory, five-step tool cap, and no automatic model retry remain intact.

Explicit non-goals

  • No attachments, conversation sharing, history search, model selector, authentication, or new AI tools.
  • No Prisma schema migration. Historical tool UI will reconstruct from the existing toolCalls and toolCallId fields.
  • No light theme in this plan.
  • No replacement of tRPC, Prisma, AI SDK, React Markdown, or the existing provider.
  • No claim that a saved memory was used in a response unless the server records that fact. The context panel will say “Saved memories,” not “Memories used.”

Approval and success criteria

  • Nocturne Studio visual identity is consistent across chat, navigation, tools, feedback, and settings.
  • All existing product flows and ownership boundaries remain intact.
  • Regenerate never duplicates the last user message and failed regeneration does not destroy the stored response.
  • Historical tool activity looks equivalent to live tool activity after refresh.
  • No critical action depends on hover.
  • Keyboard focus is visible and restored after sheets close.
  • Normal text meets 4.5:1 contrast; UI boundaries meet 3:1 where required.
  • Interactive targets are at least 44×44px on touch layouts.
  • Long prompts, code, tables, and tool output cannot break the mobile layout.
  • Reduced-motion preferences are honored globally.
  • Tests, typecheck, and production build pass with fresh evidence.
  • The remaining upstream live checks retain truthful pending/pass status.
